# Coding-Test-Practice-Python

1. 문제를 정확히 이해하고
2. 규칙성을 발견하여(아이디어)
3. 이를 구현한다.

# 문제풀이 연습 단계
**1. 문제 읽기**
**2. 접근 방법 구상**
**3. 핵심코드 손코딩**
4. 코드구현
5. 디버깅 및 개선

# 1. 문제 읽기
1. 빠르게 읽기
2. Test case를 손으로 풀면서 읽기(주의사항, 제약조건 적어놓기)
3. 100% 정답이 안나올 경우, 한번 더 꼼꼼히 읽기

# 2. 접근 방법 구상
1. 완전히 새로운 문제는 없다!
* 이전에 풀었던 문제와 유사한지
* 특정 자료구조를 적용하면 문제 설명과 비슷한지
* 전형적인 알고리즘 문제인지
2. 문제의 **설명대로** 시도
* 문제의 설명대로 Test Case의 입력과 출력대로 최대한 손으로 풀어보기
* 손으로 풀어보면 규칙성을 발견할 수 있음
3. 규칙성을 발견하기 힘들다면
* 가능한 모든 경우를 직접 처리해서 정답을 도출하기(시간, 메모리 제약 조건에 걸릴 수도 있음)
* 일부분으로 쪼개서 살펴보기
* 반대로 접근해보기
    이익 -> 손해
    등산 -> 하산
4. 모든 구상을 시각적으로(**손**)

# 3. 핵심코드 손코딩
시각적으로 보이게 적기(그리기)
* 범수나 반복문의 범위
* 핵심 수식
* 변수
* 지도

# 4. 코드구현
손코딩 한 것을 실수하지 않고 typing

# 5. 디버깅 및 개선
다른 방법, 알고리즘을 적용해서 풀 순 없을까?

# 문제풀이 연습 팁
1. 기본기는 철저히 연습(**틀, 루프, 범위**)
    1) 기본기가 탄탄해야 구상한 아이디어 실수 없이 구현
    2) **정확한 입출력**, 실수 없는 **2차원 배열 처리** 및 다중 루프구현
    3) 가장 효율적인/짧은/멋있는 코드보다 **기본적인 [반복/조건]**을 빈틈없이 구현(Test Case를 빈틈없이 처리할 수 있어야 함)
    4) 손코딩: 사용하는 주요 array, **범위**, 핵심코드를 **실명/시각적으로 설계**하고 접근
    5) 필요한 위치에서 필요한 디버깅 가능하도록 **디버거 사용법 숙지**
    6) 문제읽기->접근방법 구상->핵심코드 손코딩(키보드를 멀리하세요)
2. 나만의 환경/루틴으로 심리적 안정
    1) 파이참 환경, 폰트, 폰트크기, 창 배치, TC 입력파일, A4용지, 풀이 순서
    2) 익숙한 이름: 입력 받는 변수, 선언한 변수, 특정용도 사용 변수 등
    3) 함수호출, 조건, 반복, break, continue 등 익숙한 방법으로 구현

**아무리 디버깅을 해봐도 숨겨진 TC를 통과할 수 없을 때: 시간이 60% 남았을 때, 전체 코드를 과감히 지우고 다시 6) 문제읽기->접근방법 구상->핵심코드 손코딩으로 돌아간다**

# 엣지 케이스

* 배열: 최솟값, 최댓값으로 값을 모두 맞춰서 풀어보기, 최소 크기, 최대 크기
* 정렬: 이미 정렬된 케이스, 역순 정렬된 케이스 만들어서 풀어보기

* 한 번도 변화가 안 일어나거나
* 첫 턴에 끝나거나
* 마지막 턴까지 가거나
* 동시에 여러 이벤트가 발생하는 경우

* 손코딩 할 때: 문제 조건에서 확인 가능한 시나리오 + 테스트 케이스에서 확인 가능한 시나리오 + 또 다른 시나리오는 없나 최대한 고민해서 TC를 **새로 만들어서** 하기
